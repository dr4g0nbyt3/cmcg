CMCG // Connected Media Container & Generator

A dynamic video container format where embedded media assets — images, clips, overlays — are resolved from live URLs or local variables at render time, enabling a single video template to display different content without re-encoding.

01 — Overview

What is CMCG?

CMCG (Connected Media Container & Generator) is a dynamic video template format. Think of it like an HTML page for video — the structure, timing, and layout are fixed, but the media slots inside it can be populated from URLs, local files, or runtime variables at the moment of playback or render.

A .cmcg file is essentially a JSON/manifest container that describes a video timeline. Each "slot" in that timeline points to a media source — a URL, a local file path, or a named variable. When played or exported, the runtime fetches those sources and composites them into the final video.

💡

Simplest analogy: It's a video template. Like a billboard that you can swap the ad image on without reprinting the whole board — except the "billboard" is a video and the swapping happens automatically via URL or variable.

02 — Core Concept

How The Media Fetching Works

Each media slot in a CMCG template has a source resolver. The resolver checks a priority chain:

Local Variablee.g. $myImage
→
Local File Path/assets/ad.png
→
Remote URLhttps://cdn.../img.jpg
→
Fallback Assetplaceholder.png

This means the same template can serve a local ad when running offline, a remote ad when connected, or a dynamic ad when a variable is passed in at runtime (e.g. from your app, a URL parameter, or a script).

Example Slot Definition

// Inside a .cmcg manifest — one media slot { "id": "slot_ad_banner", "type": "image", // or "video", "overlay", "text" "start": 2.0, // seconds into video timeline "duration": 8.0, "position": { "x": 40, "y": 620, "w": 1840, "h": 200 }, "source": { "variable": "$adImage", // check variable first "local": "./assets/ad.png", // fallback to local file "remote": "https://cdn.example.com/ads/current.jpg", "fallback": "./assets/placeholder.png" }, "cache": "session", // none | session | permanent "fetchTimeout": 3000 // ms before falling back }
03 — File Format

The .cmcg File Structure

A .cmcg file is a ZIP-like bundle (similar to .docx or .apk) containing:

📄
manifest.json

The full timeline definition — slots, timing, resolution, frame rate, audio tracks, and variable declarations.

🖼️
/assets/

Optional embedded local fallback assets — images, audio clips, video segments that travel with the file.

🎞️
/base/

The base video layer(s) — static background video, intro/outro sequences, static overlays that never change.

⚙️
config.json

Runtime settings — default variables, fetch behavior, MP4 export resolution, render quality presets.

Full Manifest Example

{ "cmcg_version": "1.0", "meta": { "name": "Summer Ad Template", "resolution": [1920, 1080], "fps": 30, "duration": 30.0 }, "variables": { "$adImage": null, // passed at runtime "$brandColor": "#ff6b6b", "$logoUrl": null }, "base_video": "./base/background.mp4", "slots": [ { /* slot_ad_banner defined above */ }, { "id": "slot_logo", "type": "image", "start": 0, "duration": 30, "source": { "variable": "$logoUrl", "local": "./assets/logo.png" } } ] }
04 — Variable System

How Variables Work

Variables are the power feature. They allow external systems — websites, apps, scripts — to inject content into a CMCG template without touching the file itself.

Variable Type How It's Set Use Case
$localVar Hardcoded in config.json or at launch Offline use, kiosk displays
$urlParam Passed as a query parameter to a web player ?adImage=https://... in browser URL
$envVar Environment variable at CLI render time Server-side batch rendering pipeline
$apiVar Fetched from a JSON endpoint at play time Dynamic ad network, real-time content
$scriptVar Injected via JS if using the web runtime App integration, A/B testing
🔌

API Variable Example: A slot can point its $adImage variable to a JSON endpoint that returns {"imageUrl": "https://..."} — so every time the video plays, it fetches the latest ad image. Perfect for ad rotation, seasonal content, or personalization.

05 — MP4 Export

Saving / Exporting as .mp4

A CMCG file is a live/dynamic format. But you can bake it into a static .mp4 at any time. The renderer fetches all remote sources, resolves all variables, composites the layers, and encodes to H.264/H.265 video.

▶
Live Playback (.cmcg)

Dynamic. Fetches fresh media every time. Variables can differ per session. Requires the CMCG runtime player.

dynamiclive URLsswappable
💾
Baked Export (.mp4)

Static snapshot. All sources resolved at export time. Plays anywhere. No runtime needed.

portableshareableuniversal

Export CLI (Planned)

# Render with variable overrides, export to mp4 cmcg render ./my-template.cmcg \ --var adImage=https://cdn.mysite.com/ads/july.jpg \ --var logoUrl=https://cdn.mysite.com/logo.png \ --output ./output/july-ad.mp4 \ --quality high \ --resolution 1920x1080

Export From Web Player (Planned)

The web-based CMCG player will have an Export to MP4 button. It uses FFmpeg.wasm (WebAssembly port of FFmpeg) to render in-browser — no server required for simple exports.

08 — Market Landscape

Competitive Analysis & Market Gap

The dynamic creative optimization (DCO) market is projected to exceed $4B globally by 2027, with DCO-driven ads reporting up to 30% higher CTRs and 20% better conversion rates versus static. Yet every major player has the same architectural limitation: they treat video as a monolith.

How DCO Works Today

Platforms like Google DV360 and Meta Ads Manager do asset selection or overlay swaps — they pick from pre-made assets or layer text/images on a static base video. The base video itself never changes. Google's Director Mix lets you swap bumper elements on YouTube but caps at 15 video assets per campaign. Meta removed standalone DCO in 2023 and only shows individual asset performance, not which combinations worked.

⚠️

The production bottleneck: Teams need 10–50+ variations per month for effective testing, but only 5–10% of creatives become scalable performers. Every variant requires a full re-render. This is the problem CMCG eliminates.

Competitive Landscape

Player Approach Limitation
Google Director Mix / PMax Overlay swaps on YouTube; asset selection Walled garden; capped at 15 assets; no in-video composition
Meta Flexible Ads Algorithm picks asset combinations No per-placement control; opaque combination reporting
Celtra / Clinch Creative automation & real-time DCO Video is overlay-only, not compositional
Innovid (CTV-first) Dynamic CTV ad serving with DCO Closest competitor; still template-overlay model
Waymark AI generates entire videos from brand assets Creates new videos, no runtime slot swapping
Synthesia / HeyGen AI avatar-based video generation Talking-head only; not for product/ad creative

🎯

The gap CMCG fills: No player offers a standardized, open container format for video where slots resolve at serve time without re-rendering. Everyone either pre-renders all variants, overlays on static video, or generates entirely new videos with AI. CMCG is architecturally closer to how HTML works — and that's the unlock.

09 — AI Integration Strategy

AI as the Runtime Decision Engine

86% of advertisers use or plan to use generative AI for video ad creative, with GenAI expected to power 40% of all video ad creative by 2026. AI-personalized video ads show 6–9 percentage point CTR improvement over both personalized image ads and generic video. The industry is shifting toward agentic AI — systems that autonomously plan, decide, and act.

AI today generates whole new videos or picks from pre-made assets. There is no widely adopted system where AI makes granular, slot-level decisions within a single video template at serve time. This is CMCG's platform play: the template is the creative scaffolding, AI is the runtime decision engine that fills it.

Slot-Level Decisions AI Can Make

🖼️
Visual Slots

Select which product image fills the hero slot. Choose spokesperson clip from a library. Adjust color theme to match brand, season, or content genre. Pick background scene variant.

✍️
Text & CTA Slots

Choose CTA text variant based on performance data. Swap headline copy per audience segment. Adjust offer text based on funnel stage (awareness → consideration → conversion).

🎵
Audio Slots

Select background music mood matching content being watched. Swap voiceover language for regional targeting. Choose hype track per event or audience energy level.

🏷️
End Card Slots

Select end-card offer based on viewer's funnel stage. Choose which testimonial clip to insert. Pick store badge (App Store vs. Google Play) based on device.

Data Signals (Privacy-Safe)

The AI decision layer uses contextual and first-party signals only — no cross-site behavioral tracking required:

time of day weather device type content genre geographic region purchase history (1P) loyalty tier (1P) browsing category (1P) slot combo performance loops creative harmony scoring attention heatmap predictions
🧠

The feedback loop: AI learns which slot combinations drive higher completion rates, CTR, and conversions per audience segment over time. Unlike Meta's opaque algorithm, CMCG's slot-level architecture makes every decision transparent and auditable — advertisers can see exactly which hero image + CTA + end card combination won.

10 — User Experience & Personalization

Relevance Without Surveillance

Research shows a clear tension in personalized advertising. The opportunity is massive — but the execution determines whether users engage or recoil.

📈
The Upside

DCO-driven ads achieve ~30% higher CTR and 20% better conversions vs. static. AI-personalized video ads show 6–9 percentage point CTR improvement. Consumers prefer ads that match their context and interests.

😬
The Downside

60% of consumers report discomfort with AI-driven personalization. 30% unsubscribe because ads feel "creepy." Each 1-point increase in reactance on a 7-point scale reduces purchase intent by half a point.

The mechanism is well-understood: perceived surveillance drives creepiness. Retargeting someone with the exact product they browsed 30 seconds ago feels invasive. But swapping a hero image to match their region, season, or the content they're watching feels helpful.

🎯

CMCG's sweet spot: Slot-based personalization enables contextual personalization — location, time of day, weather, content genre, device — without requiring invasive behavioral tracking. The ad feels relevant because it matches the context, not because it stalked the user across the internet. This is the personalization model that research shows users actually welcome.

Vendor Value Proposition

  1. Eliminate the re-encode bottleneck. One template, infinite variants at serve time. No render farm required for personalization.

  2. Transparent creative intelligence. Unlike Meta's opaque algorithm, every slot decision is auditable. Advertisers see exactly what won and why.

  3. Cross-device continuity. Same .cmcg template plays on CTV, mobile, and web — slots resolve per device context without maintaining separate campaigns.

  4. Privacy-forward by architecture. Contextual signals replace behavioral tracking. Ready for a cookieless, privacy-regulated future.

12 — Resolved Decisions

Design Decisions Locked

The following questions have been answered and are now considered spec-level decisions for v1.

📦
File Format — Custom Binary (End Goal)

Decision: Start with ZIP for simplicity in v0.1, but the end-game target is a custom binary format. ZIP gives you fast iteration and free tooling. Once the spec stabilizes, migrate to a purpose-built binary that is more compact, faster to parse, and harder to reverse-engineer casually.

The binary format also opens the door to better DRM integration at the byte level — something ZIP can't cleanly support.

v0.1: ZIPv1.0+: Custom Binary
🔐
DRM / Template Protection — Deferred, Architecture Reserved

Decision: No hard stance yet, but the architecture must leave room for it. When building the binary format, reserve an encryption header block so DRM can be bolted on later without breaking the spec. This way the decision doesn't have to be made now but the option isn't closed off.

If the template marketplace becomes real, DRM becomes essential — creators need to know their base video can't just be unzipped and extracted.

encryption header reservedrevisit at Phase 4
🔒
Remote Fetch Security — HTTPS-Only + Allowlist

Decision: Best balance of security and web usability is HTTPS-only enforcement (no plain HTTP sources ever) combined with an optional per-template domain allowlist. If no allowlist is defined, any HTTPS source is permitted. If one is defined, only those domains can be fetched.

CORS is handled at the browser level naturally for the web player. The CLI renderer will enforce HTTPS at the fetch layer in Rust. This keeps it secure by default without being so restrictive it breaks real-world use cases.

HTTPS-only enforcedallowlist optionalCORS browser-native
💾
Caching Strategy — User-Configurable

Decision: Caching is user-configurable per slot, with sensible defaults. Three modes supported:

none — always fetch fresh (live ads, real-time content). session — cached for the duration of one playback session (default). permanent — cached locally until manually cleared (offline-friendly templates).

The manifest can set a default cache mode globally, and individual slots can override it. This gives template creators full control without forcing one-size-fits-all behavior.

nonesession (default)permanentper-slot override
🎵
Audio Slots — YES, Included in v1

Decision: Dynamic audio slots are in scope for v1. A slot type of "audio" works identically to image/video slots — it has a start time, duration, and a source resolver chain (variable → local → remote → fallback). This enables swapping background music tracks, voiceover clips, or sound effects via URL.

This unlocks huge value for church/ministry use (swap the worship song each week), esports (swap the hype track per event), and ads (regional audio/language swaps without re-encoding the video).

type: "audio" confirmedv1 scopemusic · voiceover · SFX
💰
Monetization — Open Format + Commercial Layer

Decision: CMCG is built as an open format (published to GitHub, BSL licensed) with a commercial layer on top. The spec and reference player are open so the format can grow into a standard. Revenue comes from the template marketplace, the batch render API, and the visual editor GUI — all of which stay proprietary.

Primary early markets: churches & ministries (Bearded Digital client overlap), esports orgs (Toxic Tournaments overlap), and small ad agencies who need personalized video at volume without a full production team.
